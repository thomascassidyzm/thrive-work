1. Introduction and Purpose

ClearMinds is committed to protecting the confidentiality, integrity, and availability of all information assets, particularly personal data and sensitive health information entrusted to us by service users and corporate partners.

This Information Security Policy establishes the framework for managing information security risks and ensuring compliance with legal, regulatory, and contractual requirements, including:

  • UK General Data Protection Regulation (UK GDPR)
  • Data Protection Act 2018
  • Network and Information Systems (NIS) Regulations 2018
  • Professional clinical information governance standards
  • Cyber Essentials / Cyber Essentials Plus certification requirements [IF APPLICABLE]
  • ISO 27001 Information Security Management standards [IF PURSUING]

2. Scope and Application

This policy applies to:

  • All employees, contractors, consultants, and temporary workers
  • All information in any form (electronic, paper, verbal)
  • All information systems, networks, and devices
  • All locations where ClearMinds operates
  • Third-party service providers processing information on our behalf

4. Information Security Principles

Confidentiality

Information is accessible only to those authorised to access it. Protection against unauthorised disclosure.

Integrity

Information is accurate, complete, and protected against unauthorised modification or destruction.

Availability

Information and systems are accessible and usable when needed by authorised users.

6. Risk Management

6.1 Risk Assessment

ClearMinds conducts regular information security risk assessments to identify, analyse, and evaluate risks to information assets:

  • Annual Comprehensive Assessment: Full review of all information security risks
  • Quarterly Reviews: Review of high-risk areas and recent incidents
  • Ad-hoc Assessments: For new systems, projects, or significant changes
  • Data Protection Impact Assessments (DPIAs): For high-risk data processing

6.2 Risk Treatment

Identified risks are treated through:

  • Risk Reduction: Implementing controls to mitigate risks
  • Risk Avoidance: Eliminating activities that create unacceptable risks
  • Risk Transfer: Insurance or contractual transfer to third parties
  • Risk Acceptance: Accepting low-level risks with senior management approval

6.3 Risk Register

ClearMinds maintains a comprehensive risk register documenting:

  • Identified information security risks
  • Risk ratings (likelihood and impact)
  • Existing controls
  • Treatment plans and actions
  • Risk owners and review dates

7. Technical Security Controls

7.1 Network Security

  • Firewall Protection: Next-generation firewalls at network perimeter
  • Network Segmentation: Separation of production, development, and management networks
  • Intrusion Detection/Prevention: 24/7 monitoring for malicious activity
  • VPN: Encrypted virtual private network for remote access
  • Wireless Security: WPA3 encryption for Wi-Fi networks
  • DDoS Protection: Distributed denial of service mitigation

7.2 Endpoint Security

  • Anti-virus/Anti-malware: Enterprise-grade protection on all devices
  • Endpoint Detection and Response (EDR): Advanced threat detection
  • Device Encryption: Full disk encryption (BitLocker/FileVault) on all devices
  • Automatic Updates: Operating system and application patching
  • Device Management: Mobile Device Management (MDM) for mobile devices
  • Screen Lock: Automatic lock after inactivity

7.3 Data Protection

  • Encryption in Transit: TLS 1.3 minimum for all data transmission
  • Encryption at Rest: AES-256 encryption for stored data
  • Data Loss Prevention (DLP): Monitoring and prevention of unauthorised data transfer
  • Email Security: SPF, DKIM, DMARC implementation; email encryption for sensitive data
  • Secure File Transfer: SFTP/FTPS for file transfers
  • Database Security: Database encryption, access controls, and audit logging

7.4 Application Security

  • Secure Development: OWASP guidelines for application development
  • Code Review: Security code review before production deployment
  • Vulnerability Scanning: Regular automated vulnerability assessments
  • Penetration Testing: Annual third-party penetration testing
  • Web Application Firewall (WAF): Protection against web-based attacks
  • API Security: Authentication, rate limiting, and encryption for APIs

7.5 Cloud Security

  • UK-based Hosting: Primary data hosting within the United Kingdom
  • ISO 27001 Certified Providers: Use of certified cloud service providers
  • Data Sovereignty: Ensuring data remains under UK jurisdiction
  • Encryption: Cloud provider encryption plus application-level encryption
  • Access Controls: Identity and access management (IAM) policies
  • Regular Audits: Third-party security audits of cloud providers

7.6 Backup and Recovery

  • Regular Backups: Daily incremental, weekly full backups
  • Encrypted Backups: All backups encrypted at rest
  • Offsite Storage: Geographically separate backup storage
  • Backup Testing: Quarterly restore testing
  • Retention: Backups retained per Data Retention Policy
  • Recovery Time Objective (RTO): 4 hours for critical systems
  • Recovery Point Objective (RPO): 24 hours maximum data loss

8. Access Control

8.1 User Access Management

  • Unique User IDs: Individual accounts for each user (no shared accounts)
  • Strong Authentication: Complex password requirements (12+ characters, complexity rules)
  • Multi-Factor Authentication (MFA): Mandatory for all remote access and administrative access
  • Role-Based Access Control (RBAC): Access based on job role and need-to-know
  • Principle of Least Privilege: Minimum access necessary to perform duties
  • Privileged Access Management (PAM): Enhanced controls for administrative accounts

8.2 Access Review

  • Quarterly Reviews: Review of user access rights
  • Joiner/Mover/Leaver Process: Access provisioned/changed/removed promptly
  • Immediate Revocation: Access removed immediately upon termination
  • Dormant Account Monitoring: Accounts inactive for 90+ days disabled

8.3 Password Management

  • Minimum 12 characters length
  • Complexity requirements (upper/lower case, numbers, special characters)
  • Password expiry: 90 days for standard users, 60 days for privileged accounts
  • Password history: Cannot reuse last 5 passwords
  • Account lockout: 5 failed attempts results in temporary lockout
  • Password manager recommended for secure password storage
  • Never store passwords in plain text or share passwords

8.4 Remote Access

  • VPN required for all remote access to corporate systems
  • MFA mandatory for remote access
  • Company-issued devices preferred for remote access
  • BYOD (Bring Your Own Device) permitted only with MDM enrollment and compliance
  • Remote desktop connections encrypted and monitored

9. Physical and Environmental Security

9.1 Physical Access Control

  • Controlled Access: Keycard/biometric access to premises
  • Visitor Management: All visitors signed in, escorted, and badged
  • CCTV Monitoring: Security cameras at entry points (retained 30 days)
  • Secure Areas: Additional controls for server rooms and records storage
  • Out of Hours Access: Logged and reviewed

9.2 Equipment Security

  • Asset Register: All IT assets tracked and logged
  • Equipment Labelling: Clear ownership and classification labels
  • Secure Storage: Locked cabinets for portable devices when not in use
  • Disposal: Secure disposal of equipment (wiping/destruction) with certificates
  • Off-site Equipment: Approval required; encryption mandatory

9.3 Clear Desk and Clear Screen

  • No sensitive documents left on desks unattended
  • Documents locked away at end of day
  • Screens locked when leaving workstation
  • Automatic screen lock after 10 minutes inactivity
  • Confidential waste bins for sensitive documents

9.4 Environmental Controls

  • Fire suppression systems in server rooms
  • Temperature and humidity monitoring
  • Uninterruptible Power Supply (UPS) for critical systems
  • Regular testing of environmental controls

10. Operational Security

10.1 Change Management

  • Formal change control process for all system changes
  • Testing in non-production environment before production deployment
  • Change approval by IT manager or above
  • Rollback plans for all changes
  • Documentation of all changes

10.2 Capacity Management

  • Regular monitoring of system capacity and performance
  • Forecasting future capacity requirements
  • Scalability planning for growth

10.3 Malware Protection

  • Anti-virus/anti-malware on all endpoints and servers
  • Automatic signature updates
  • Real-time scanning and heuristic detection
  • Email attachment scanning
  • Web filtering to block malicious sites
  • User education on recognising phishing and malware

10.4 Logging and Monitoring

  • Centralized Logging: Security Information and Event Management (SIEM) system
  • Audit Trails: Access to sensitive data logged and retained
  • Log Retention: Security logs retained for 2 years
  • Real-time Monitoring: 24/7 security monitoring
  • Alerting: Automated alerts for security events
  • Regular Review: Weekly review of security logs

10.5 Vulnerability Management

  • Patch Management: Critical patches applied within 14 days, high-risk within 30 days
  • Vulnerability Scanning: Monthly automated scans
  • Penetration Testing: Annual third-party penetration tests
  • Security Advisories: Monitoring vendor security bulletins
  • Remediation Tracking: Tracking and verification of vulnerability fixes

12. Third-Party Security

12.1 Supplier Due Diligence

Before engaging third-party suppliers who will process or access ClearMinds information:

  • Security assessment questionnaire completed
  • Review of supplier's security certifications (ISO 27001, Cyber Essentials, etc.)
  • Assessment of data protection compliance
  • Financial and reputational due diligence
  • Right to audit provisions in contracts

12.2 Contractual Requirements

  • Data Processing Agreements (DPAs) for all processors
  • Confidentiality and non-disclosure agreements
  • Security requirements and standards specified
  • Incident notification obligations
  • Right to audit and inspect
  • Liability and indemnity provisions
  • Secure data return/deletion upon termination

12.3 Ongoing Monitoring

  • Annual review of supplier security posture
  • Review of supplier security incidents
  • Re-assessment upon contract renewal
  • Audits where appropriate

16. Acceptable Use

16.1 Acceptable Use of IT Resources

ClearMinds IT resources must be used responsibly and for legitimate business purposes:

  • Primary use for business purposes; limited personal use permitted
  • No illegal, offensive, or inappropriate content
  • No unauthorized software installation
  • No circumventing security controls
  • No sharing of user credentials
  • Respect for intellectual property rights

16.2 Email and Internet Use

  • Email and internet use monitored for security purposes
  • No expectation of privacy for business communications
  • Care when opening attachments or clicking links
  • Report suspicious emails immediately
  • Use encryption for sending sensitive data via email

16.3 Social Media

  • No posting of confidential or sensitive ClearMinds information
  • Professional conduct expected on social media
  • Clear separation between personal and professional accounts
  • Social media policy applies

17. Mobile Devices and Remote Working

17.1 Mobile Device Security

  • Company devices: Full encryption, MDM enrollment, remote wipe capability
  • Personal devices (BYOD): MDM enrollment required for corporate data access
  • Strong passcode/biometric authentication required
  • Automatic lock after inactivity
  • Lost/stolen devices reported immediately
  • Public Wi-Fi: VPN required

17.2 Remote Working

  • VPN required for accessing corporate systems remotely
  • Secure home working environment (privacy, secure Wi-Fi)
  • Family members should not access company devices
  • Video calls: Care with background and environment
  • Document security: Secure storage at home, no leaving in vehicles

18. Cryptography

18.1 Encryption Standards

  • Data in Transit: TLS 1.3 minimum
  • Data at Rest: AES-256 encryption
  • Email Encryption: S/MIME or PGP for sensitive emails
  • File Encryption: Encrypted zip or dedicated encryption software
  • Full Disk Encryption: BitLocker (Windows) or FileVault (Mac)

18.2 Key Management

  • Centralized key management system
  • Regular key rotation
  • Secure key storage (Hardware Security Modules where appropriate)
  • Key backup and escrow procedures
