Smart Start Minds Partnership

1. Executive Summary

This document demonstrates ClearMinds B2B's integration with a CQC-regulated clinical mental health service provider, Smart Start Minds, ensuring a seamless pathway from wellness support to professional clinical care when needed.

Purpose of this Partnership Documentation:

  • Demonstrate integration with CQC-regulated clinical services
  • Ensure seamless wellness-to-clinical pathway for users
  • Provide quality assurance through regulated partner oversight
  • Build insurance partner confidence in clinical safety measures

This partnership ensures that while ClearMinds focuses on wellness, resilience-building, and self-help resources, users who require clinical-level mental health support have immediate access to professional, regulated services through Smart Start Minds.

4. Clinical Referral Pathway

4.1 User Journey Overview

Users begin their journey with ClearMinds B2B's wellness platform, accessing self-help resources, resilience-building tools, and live chat support. If during their use of ClearMinds it becomes apparent they would benefit from clinical support (based on screening tool scores, live chat advisor assessment, or user self-identification), they are offered a referral to Smart Start Minds.

This ensures a seamless pathway from wellness support to regulated clinical care when needed, with no gaps in support and clear communication throughout the process.

4.2 Referral Criteria

When is a user referred to Smart Start Minds?

A referral to Smart Start Minds is appropriate when:

  • User self-identifies need for clinical support (e.g., requests therapy or psychiatric assessment)
  • User responses indicate moderate-severe symptoms (e.g., PHQ-9 score ≥10, GAD-7 score ≥10)
  • User expresses suicidal thoughts or self-harm ideation (non-emergency - for emergency situations see section 4.4)
  • User has complex mental health history requiring clinical assessment (e.g., previous diagnoses, medication use, multiple conditions)
  • Live chat advisor identifies clinical need during conversation (e.g., significant functional impairment, distress beyond wellness scope)
  • User has not responded to self-help approaches and symptoms persist or worsen
  • User presents with symptoms requiring diagnostic assessment (e.g., ADHD, OCD, PTSD)
  • User requests medication review or psychiatric consultation
  • Clinical condition identified that is outside ClearMinds B2B's wellness scope
NOTE: Emergency Situations

Emergency situations (imminent risk to self or others) are escalated directly to emergency services (999) or NHS Crisis Line (111 option 2), NOT through routine Smart Start Minds referral. See section 4.4 for full emergency protocol.

4.3 Referral Process

The referral process follows a clear, user-centered 7-step pathway:

  1. Trigger Identification: User meets referral criteria OR requests clinical support. This may occur through screening tool results, live chat interaction, or user self-referral via the platform.
  2. User Consent: User is informed about Smart Start Minds (who they are, what they provide, CQC regulation, what to expect) and consents to:
    • Referral to Smart Start Minds
    • Sharing of relevant information (name, contact details, symptom scores, assessment responses, relevant history)
    • Feedback sharing between SSM and ClearMinds (for continuity of care)
    Consent is documented and stored securely.
  3. Information Gathering: ClearMinds collects relevant information to support the referral:
    • User demographics (name, contact details, employer if relevant)
    • Presenting concerns and symptoms
    • PHQ-9, GAD-7, and other screening tool scores
    • Brief mental health history (if provided)
    • Current support/treatment (GP, medication, etc.)
    • User preferences (appointment times, therapy type if known)
    • Any risk factors or safeguarding concerns
  4. Secure Transfer: Information is securely shared with Smart Start Minds via encrypted email to dedicated referral inbox (referrals@smartstartminds.co.uk). All transfers comply with GDPR and information governance standards. Referral form template used (see Appendix D).
  5. SSM Contact: Smart Start Minds contacts user within 2 working days to arrange assessment. Initial contact made via user's preferred method (phone/email). User provided with SSM direct contact details for any queries.
  6. Clinical Assessment: Smart Start Minds conducts professional assessment (psychiatric or psychological) to determine diagnosis, treatment needs, and appropriate intervention. Assessment conducted by qualified clinician (Consultant Psychiatrist or Clinical Psychologist).
  7. Outcome: Following assessment, one of three outcomes:
    • User enters SSM care: Treatment plan developed (therapy, medication, or combination). User begins treatment with SSM.
    • User advised to continue with ClearMinds: No clinical-level need identified. User supported to continue using ClearMinds's wellness resources.
    • User signposted elsewhere: Needs better met by another service (e.g., GP for physical health, specialist NHS service, crisis team). SSM provides appropriate signposting and information.

4.4 Emergency Situations

EMERGENCY PROTOCOL

If a user presents with IMMINENT RISK to themselves or others:

  1. DO NOT use routine Smart Start Minds referral process
  2. Immediately direct user to emergency services:
    • 999 - Emergency services (ambulance, police)
    • 111 option 2 - NHS Mental Health Crisis Line
    • Local A&E department
  3. Provide crisis contact information:
    • Samaritans: 116 123 (24/7)
    • Crisis Text Line: Text SHOUT to 85258
    • Papyrus (under 35s): 0800 068 4141
  4. Stay with user (if in-person or on live chat) until emergency support confirmed
  5. Notify Smart Start Minds for follow-up coordination (if user consents and appropriate)
  6. Document incident per ClearMinds's incident reporting process
  7. Debrief and support for staff member involved

Staff Training: All ClearMinds staff (especially live chat advisors) are trained to recognize emergency situations, follow this protocol, and provide appropriate immediate response. Regular training refreshers and scenario practice ensure competence.

8. User Experience & Support

The user experience is central to the ClearMinds B2B-Smart Start Minds partnership. Every effort is made to ensure a seamless, supportive transition when clinical care is needed.

User-Centered Principles

  • Transparency: Users always informed when transitioning between services, with clear explanation of why referral is being offered
  • Continuity of support: Users can continue using ClearMinds's wellness resources while receiving Smart Start Minds care (if clinically appropriate)
  • Clear communication: Users understand they are engaging with a different organization (Smart Start Minds), with a different team, providing clinical (not wellness) support
  • User choice: Referral to Smart Start Minds is offered, not mandatory. Users can decline and continue with ClearMinds or seek alternative support
  • Feedback welcomed: Users can provide feedback on referral experience to both organizations, informing continuous improvement

User Journey Support Points

Stage Support Provided
Pre-referral
  • Information about Smart Start Minds (who they are, CQC regulation, services)
  • What to expect from assessment and potential treatment
  • Consent process explanation (what data shared, why, user rights)
  • Option to ask questions before deciding on referral
  • User Information Sheet provided (Appendix E)
Referral
  • Confirmation of referral sent to user
  • Timeline expectations clearly communicated
  • Smart Start Minds contact details provided
  • Reassurance that ClearMinds support remains available
Waiting for SSM contact
  • Continued access to ClearMinds wellness resources
  • Live chat support available if needed
  • Reminder of crisis contacts if urgent concerns arise
  • Option to contact ClearMinds if SSM contact not received within expected timeframe
During SSM care
  • ClearMinds wellness resources remain accessible (unless clinically contraindicated - rare)
  • Users can continue using self-help content, mood tracking, etc. as adjunct to SSM treatment
  • Coordination between services if user has questions about which service to contact for what
Post-SSM care (discharge/completion)
  • SSM provides discharge summary to user (and to ClearMinds with consent)
  • ClearMinds offered as ongoing wellness support resource
  • Relapse prevention planning may include ClearMinds resources
  • Re-referral pathway available if clinical needs re-emerge

Communication with Users

Clear, compassionate communication is maintained throughout:

  • Plain language used (avoiding clinical jargon)
  • Cultural sensitivity and accessibility considerations
  • Multiple communication channels available (email, phone, live chat)
  • Reasonable adjustments made for users with accessibility needs
  • Interpreting services available if needed (through SSM)

User Feedback Mechanisms

  • Post-referral feedback survey (ClearMinds) - captures experience of referral process
  • Post-assessment feedback (SSM) - captures first impressions of SSM service
  • End-of-treatment feedback (SSM) - captures overall experience and outcomes
  • Ongoing feedback options - users can provide feedback at any time via either service
  • Annual user experience review - anonymized themes shared between organizations
