Key Insight

Thrive is a platform, not a single product. The assessment layer is the core IP that intelligently routes employees to the right combination of content, coaching, and clinical services.

Employee Journey - From Invitation to Ongoing Engagement

Complete flow showing how an employee moves through the Thrive platform, from initial assessment to quarterly check-ins.

1

Invitation & Landing

Employee receives invitation → Landing page with privacy notice → Consent & account creation

2

Neuroindicator (5-10 min)

Triage assessment determines risk level: Crisis (2-3%) | Clinical (10-15%) | Wellness (85-90%)

3

OCEAN Assessment (15-20 min)

For wellness pathway: 60-80 questions profiling personality traits

4

Home→Work Influence (2 min)

6 questions about childcare, eldercare, financial stress, etc.

5

Results & Recommendations

OCEAN profile generated → Personalized content, coaching, and clinical recommendations

6

Dashboard Access

Employee sees results, accesses content library, tracks progress

7

Quarterly Pulse (2-3 min)

5 questions to detect deterioration → May trigger clinical escalation

Total Time Commitment

Initial: 25-30 minutes (Neuroindicator + OCEAN + Home→Work)
Quarterly: 2-3 minutes (Pulse check)

Build Timeline - Three Possible Approaches

Different strategies for building the platform, with varying timelines and deliverables.

A

Option A: Content-First (3 weeks)

Deliverable: All assessment JSONs + demos (portable, reusable)

Work: Neuroindicator schema + demo | OCEAN 60-80 questions | Home→Work schema + demo | Quarterly pulse

Pros: Portable, no tech stack decision needed yet
Cons: Doesn't demo full B2B value

B

Option B: Thin Slice End-to-End (8 weeks)

Deliverable: Working demo of complete employee + employer journey

Work: Assessments (minimal viable) | Backend API + database | Employee dashboard | Employer dashboard

Pros: Complete story for Towergate
Cons: Longer timeline

C

Option C: Employer Dashboard Focus (6 weeks)

Deliverable: HR-facing dashboard showing ROI

Work: Minimal assessments | Backend (aggregation logic priority) | Impressive employer dashboard | Mock employee data

Pros: Fast to Towergate value prop
Cons: Can't show employee experience

Recommended Approach

Start with Option A (content-first) to build portable assessment library, then move to Option B for full end-to-end demo. This gives us reusable assets while building toward complete Towergate pitch.
